왜 HTML로 발표하나요?

  • 플랫폼 독립성: 브라우저만 있으면 OK!
  • 높은 자유도: 상상하는 모든 디자인과 인터랙션 구현
  • 버전 관리: Git으로 체계적인 이력 관리
  • 웹 콘텐츠 내장: 유튜브, 인터랙티브 차트 등 자유롭게 삽입
  • 쉬운 공유: URL 하나로 간편하게 공유

1단계: 기획 및 구조화

발표의 뼈대를 만드는 가장 중요한 단계

"Gemini를 브레인스토밍 파트너로 활용하세요!"

Gemini 활용 프롬프트 예시

  1. 주제/대상 정의: "나는 '주제'로 '대상'에게 발표할 거야. 핵심 목표 3가지를 정의해줘."
  2. 목차 생성: "위 목표를 바탕으로 발표 목차 초안을 작성해줘."
  3. 슬라이드별 메시지 도출: "목차 기반으로 슬라이드별 핵심 메시지를 요약해줘."
  4. 발표 대본 초안 작성: "각 슬라이드 메시지를 바탕으로 발표 대본을 써줘."

2단계: 프레임워크 선택 및 설정

처음부터 만들 필요 없이, 검증된 도구를 활용하세요.

추천 프레임워크: Reveal.js

가장 인기 있고 기능이 막강한
HTML 발표 프레임워크

기본 HTML 구조 (CDN)

<!doctype html>
<html>
	<head>
		<link rel="stylesheet" href="reveal.css">
		<link rel="stylesheet" href="theme/black.css">
	</head>
	<body>
		<div class="reveal">
			<div class="slides">
				<section>Slide 1</section>
				<section>Slide 2</section>
			</div>
		</div>
		<script src="reveal.js"></script>
		<script>
			Reveal.initialize();
		</script>
	</body>
</html>

3단계: Gemini로
콘텐츠/HTML 생성

기획 내용을 실제 슬라이드 코드로 변환

"아래 슬라이드 메시지를 Reveal.js의
<section> 태그 HTML 코드로 만들어줘.

- 슬라이드 1: ...
- 슬라이드 2: ..."

Gemini가 생성한 코드를 복사해서 붙여넣기만 하면 끝!

4단계: Gemini로
디자인/CSS 생성

나만의 맞춤 디자인 적용하기

"내 발표 자료 디자인을 커스터마이징하고 싶어.
- 전체 배경색: #222
- 강조 색상: #1E90FF
- 글꼴: 'Noto Sans KR'
위 요구사항에 맞는 CSS 코드를 작성해줘."

5단계: Gemini로
상호작용/JS 추가

HTML 발표의 꽃, 인터랙션!

"Reveal.js에서 3번째 슬라이드에 진입했을 때
'집중!'이라는 알림창을 띄우는
JavaScript 코드를 작성해줘."

정적인 발표에 동적인 재미를 더할 수 있습니다.

6단계: 배포 및 공유

완성된 발표 자료를 세상에 공개하기

추천: GitHub Pages

  1. GitHub 저장소 생성 및 코드 업로드
  2. Settings → Pages 메뉴에서 배포 설정
  3. 생성된 URL (*.github.io) 공유

전문가 팁 & 주의사항

  • 반복과 개선: 한 번에 완벽을 기대하지 말고, 대화하며 결과물 다듬기
  • 사실 확인: AI가 생성한 통계, 전문 정보는 반드시 교차 검증
  • 발표자 노트 활용: <aside class="notes"> 태그와 'S' 키
  • 오프라인 대비: 인터넷이 불안하다면 모든 파일을 로컬에 저장
