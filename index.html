프롬프트 엔지니어링이란?

"The application of science to the optimum conversion of the resources of nature to the uses of humankind."
- Encyclopedia Britannica

프롬프트 엔지니어링은 과학의 응용으로 목적에 맞게 자원을 최적화하는 것입니다.

  • 과학의 응용: 가설 수립 → 테스트 → 평가의 끊임없는 반복
  • 목적: 원하는 의도대로, 정확하고 일관된 답변을 얻는 것
  • 자원의 최적화: LLM 입출력에 드는 비용(토큰, 시간)을 최소화하는 것

LLM 음악검색 프로젝트 소개

  • 목표: 차 안에서 들리는 노래 일부만으로 원하는 곡을 정확히 찾아 정보 제공하기.
  • 주요 프로세스

    음성 입력 ➔ STT ➔ STT 결과 정규화 ➔ 가사 기반 곡 검색 ➔ 결과 제공

    • 핵심 과제: STT 출력 텍스트의 정규화 (Normalization)

      음성인식(STT) 결과물은 노이즈가 많고 불완전합니다.
      예: "배우지만단 말이양 이맘 가는 그대로 지금 내 손을 잡아봐"

    프롬프트 기법 V1: 기본 지시형

    • 특징: 해야 할 일을 명확하고 단순하게 나열하여 지시합니다.

    예시 프롬프트 (V1_AS_IS)

    
아래 입력으로 주어진 'STT 결과'로부터 3~5개의 문장을 추출하세요.
각 문장은 3~4단어로 구성되어야 합니다.
문장 간 중복이 발생해도 괜찮지만, 불필요한 조사나 불분명한 단어는 제거하세요.
문장부호(쉼표, 느낌표 등)가 있으면 적절히 제거하거나 반영하여 자연스럽게 다듬으세요.

STT 결과: "{stt_result}"

출력 형식: 각 문장을 줄바꿈(\\n)으로 구분한 형태로만 출력해주세요.

    평가 방법: 무엇을 측정할 것인가?

    좋은 프롬프트란 무엇일까요? 평가 기준을 정량적 지표와 정성적 지표로 나눌 수 있습니다.

    정량적 평가 지표

    숫자로 측정 가능한 품질 지표입니다.

    1. 성능 (Performance): 얼마나 정확하게 정답을 찾는가? (TC 성공률)
    2. 비용 (Cost): 얼마나 저렴하게 사용하는가? (평균 토큰 수)
    3. 속도 (Speed): 얼마나 빠르게 응답하는가? (평균 응답 시간)

    정성적 평가 지표

    사람이 직접 판단하는 품질 지표입니다.

    • 정확성: STT 결과의 의미를 잘 반영했는가?
    • 일관성: 반복 실행해도 결과가 유사하게 나오는가?
    • 유용성: 생성된 가사가 실제 검색에 도움이 되는가?

    테스트 방법: 어떻게 검증할 것인가?

    체계적인 검증을 위해 자동화된 테스트 파이프라인과 원칙을 수립했습니다.

    • 테스트 파이프라인 (자동화 스크립트)

      1. 미리 정의된 테스트 케이스(STT 텍스트, 정답 곡 ID)를 설정합니다.
      2. 각 프롬프트 버전(V1~V5)으로 LLM을 호출해 가사를 정규화합니다.
      3. 정규화된 가사로 실제 멜론 검색 API를 호출합니다.
      4. 검색 결과 상위 3개에 정답 곡 ID가 포함되는지 확인하여 성공/실패를 기록합니다.

    • 테스트 핵심 원칙

      - 실제 데이터 활용: 실제 발화 및 노이즈가 섞인 엣지 케이스를 포함합니다.
      - 반복 테스트: 각 프롬프트를 최소 10회 이상 테스트하여 안정성을 확인합니다.
      - 테스트 도구: Python 스크립트를 사용했지만, Promptfoo와 같은 오픈소스도 활용 가능합니다.

    주요 인사이트 (Key Insight)

    • 성능과 비용은 Trade-off 관계: 정교한 프롬프트(V5)는 성능이 높지만, 토큰 사용량과 응답 시간이 증가합니다.
